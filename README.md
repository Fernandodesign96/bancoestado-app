# BancoEstado App – Rediseño UX/UI (Proyecto Conceptual)

## Contexto
Este proyecto corresponde a un **rediseño conceptual UX/UI** de la aplicación móvil de BancoEstado, desarrollado con fines de portafolio.

El objetivo es analizar y proponer mejoras en **usabilidad, jerarquía visual y optimización de flujos**, considerando las exigencias de un entorno bancario de alta demanda.

> Este **no es un proyecto oficial de BancoEstado** ni tiene relación comercial con la institución.

---

## Problema
La aplicación presenta diversos puntos de fricción que impactan la experiencia del usuario, entre ellos:
- Jerarquía visual inconsistente
- Sobrecarga cognitiva en flujos críticos
- Poca claridad entre CTAs primarios y secundarios
- Fricción en tareas de uso frecuente (transferencias, consultas de saldo)

Estos factores afectan directamente la **eficiencia, la confianza y la percepción de estabilidad**, aspectos clave en productos fintech y bancarios.

---

## Objetivos
- Mejorar la usabilidad general de la aplicación
- Reducir fricción en flujos clave
- Optimizar jerarquía visual e información
- Aplicar principios UX y heurísticas de usabilidad
- Priorizar claridad y accesibilidad

---

## Rol
**Diseñador UX/UI**

Responsabilidades:
- Análisis UX y definición de problemas
- Optimización de flujos de usuario
- Arquitectura de información
- Diseño UI y ajuste de componentes
- Justificación de decisiones de diseño

---

## Proceso UX/UI
- Evaluación heurística (principios de Nielsen)
- Identificación de puntos de fricción
- Optimización de jerarquía y CTAs
- Consistencia visual y sistema de espaciado
- Enfoque mobile-first (390px)

---

## Solución
El rediseño propone:
- Acciones primarias más claras
- Mejor legibilidad y jerarquía
- Interacciones más simples
- Uso consistente de componentes

El foco está puesto en **usabilidad, claridad y percepción de rendimiento**, más que en cambios estéticos superficiales.

---

## Herramientas
- Figma (UX, UI, componentes)
- HTML / CSS (implementación conceptual)
- Git & GitHub (control de versiones)

---

## Aprendizajes clave
- En banca, la claridad es prioritaria frente a lo visual
- Pequeños ajustes de jerarquía reducen carga cognitiva
- UX en productos financieros debe equilibrar confianza, eficiencia y accesibilidad

---

## Disclaimer
Proyecto conceptual desarrollado exclusivamente para fines educativos y de portafolio.





# BancoEstado App – UX/UI Redesign (Conceptual)

## Context
This project is a **conceptual UX/UI redesign** of the BancoEstado mobile app, developed for portfolio purposes.

It aims to explore usability improvements, visual hierarchy, and flow optimization within a **high-demand banking environment**.

> This is **not an official BancoEstado project** and has no commercial affiliation.

---

## Problem
The original app presents several friction points that impact user experience, such as:
- Inconsistent visual hierarchy
- Cognitive overload in critical flows
- Limited clarity in primary and secondary CTAs
- Friction during high-traffic operations (transfers, balance checks)

These issues directly affect **task efficiency, trust, and perceived reliability**, which are critical in fintech and banking products.

---

## Objectives
- Improve usability and accessibility
- Reduce friction in key user flows
- Strengthen visual hierarchy and information architecture
- Align UI decisions with usability heuristics and UX principles

---

## Role
**UX/UI Designer**

Responsibilities:
- UX analysis and problem definition
- User flow optimization
- Information architecture
- UI design and component refinement
- Design rationale based on usability principles

---

## UX/UI Process
- Heuristic evaluation (Nielsen heuristics)
- Analysis of friction points in key flows
- Optimization of CTAs and hierarchy
- Component consistency and spacing system
- Mobile-first approach (390px)

---

## Solution
The redesign proposes:
- Clearer primary and secondary actions
- Improved readability and spacing
- Simplified interaction patterns
- More consistent component usage across flows

The focus was placed on **usability, clarity, and performance perception** rather than purely visual changes.

---

## Tools
- Figma (UX flows, UI design, components)
- HTML / CSS (conceptual implementation)
- Git & GitHub (version control)

---

## Key Learnings
- Designing for banking requires prioritizing clarity over aesthetics
- Small hierarchy changes can significantly reduce cognitive load
- UX decisions must balance usability, trust, and performance perception

---

## Disclaimer
This project is a **conceptual redesign created exclusively for educational and portfolio purposes**.
